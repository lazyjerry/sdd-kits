# 比較型研究報告章節模板

## 觸發時機
當任務是比較兩個以上工具、框架或專案，且需要交付可決策的研究報告時。

## 摘要
用固定章節框架輸出：安裝方法、產生檔案格式、用例、用法、差異總表、導入建議。

## 知識內容
- 建議章節順序：
  1. 專案定位摘要
  2. 安裝方法比較
  3. 產生的檔案格式與目錄結構
  4. 用例（適用情境）
  5. 用法流程（實務操作）
  6. 差異總表
  7. 導入建議
  8. 研究限制與備註
- 產出原則：
  - 比較表要含「執行器、依賴、工作單位、命令、封存/演進機制」。
  - 各章節要能對應實際落地行為（指令、路徑、文件名）。
  - 最後要給明確選型建議（何時選 A、何時選 B）。
- 本次任務案例：
  - OpenSpec vs Spec Kit 的報告即採用此模板。

### 範例
```text
A 專案：安裝 + 初始化 + 產物
B 專案：安裝 + 初始化 + 產物
→ 差異總表
→ 導入建議（以團隊成熟度/流程需求分流）
```
